# 102.2 - Instalar un gestor de arranque: Ejercicios

## Ejercicio 1
**Has modificado el archivo `/etc/default/grub` para cambiar el tiempo de espera del menu a 10 segundos. Que comando debes ejecutar a continuacion para que los cambios surtan efecto? Indica la opcion tanto para Debian como para Red Hat.**

<details>
<summary>Ver respuesta</summary>

En **Debian/Ubuntu**:
```bash
update-grub
```

En **Red Hat/CentOS/Fedora**:
```bash
grub2-mkconfig -o /boot/grub2/grub.cfg
```

Ambos comandos regeneran el archivo `grub.cfg` a partir de `/etc/default/grub` y los scripts de `/etc/grub.d/`. Sin ejecutar estos comandos, los cambios en `/etc/default/grub` no tendran efecto.
</details>

## Ejercicio 2
**Que archivo de configuracion de GRUB2 se debe editar para cambiar los parametros de arranque del kernel? Por que NO se debe editar directamente `/boot/grub/grub.cfg`?**

<details>
<summary>Ver respuesta</summary>

Se debe editar **`/etc/default/grub`**, especificamente las variables:
- `GRUB_CMDLINE_LINUX` - para parametros que se aplican a todas las entradas
- `GRUB_CMDLINE_LINUX_DEFAULT` - para parametros solo de la entrada por defecto

**No se debe editar** `/boot/grub/grub.cfg` directamente porque:
1. Es un archivo **generado automaticamente** por `grub-mkconfig` / `update-grub`.
2. Cualquier modificacion manual se **perdera** la proxima vez que se ejecute `update-grub` (por ejemplo, al instalar un nuevo kernel).
3. El archivo tiene un comentario en su cabecera indicando que no debe editarse manualmente: "DO NOT EDIT THIS FILE".
</details>

## Ejercicio 3
**Estas en el menu de GRUB y necesitas arrancar en modo de recuperacion (single user) de forma temporal. Describe los pasos exactos.**

<details>
<summary>Ver respuesta</summary>

1. En el menu de GRUB, seleccionar la entrada de Linux deseada.
2. Pulsar **`e`** para editar la entrada.
3. Buscar la linea que comienza con `linux` (o `linux16` en algunos sistemas).
4. Al final de esa linea, anadir uno de los siguientes:
   - `single` o `1` (para sistemas con SysVinit)
   - `systemd.unit=rescue.target` (para sistemas con systemd)
   - `init=/bin/bash` (para acceso directo a shell sin arrancar servicios)
5. Pulsar **Ctrl+X** o **F10** para arrancar con los cambios.

Estos cambios son **temporales** y se pierden en el siguiente reinicio.
</details>

## Ejercicio 4
**Explica la funcion del directorio `/etc/grub.d/` y describe para que sirven al menos tres de los scripts que contiene.**

<details>
<summary>Ver respuesta</summary>

El directorio `/etc/grub.d/` contiene scripts ejecutables que generan las diferentes secciones del archivo `grub.cfg`. Se ejecutan **en orden numerico** cuando se invoca `grub-mkconfig` o `update-grub`.

Scripts principales:

1. **`00_header`**: Genera la cabecera de grub.cfg con la configuracion general (timeout, entrada por defecto, resolucion grafica). Lee las variables de `/etc/default/grub`.

2. **`10_linux`**: Detecta los kernels Linux instalados en el sistema y genera las entradas del menu correspondientes (incluyendo versiones de recuperacion).

3. **`30_os-prober`**: Ejecuta `os-prober` para detectar otros sistemas operativos instalados (Windows, otros Linux) y generar entradas de menu para ellos.

4. **`40_custom`**: Permite al administrador anadir entradas personalizadas al menu de GRUB. Es el archivo que se debe editar para agregar entradas manuales.

Solo se ejecutan los scripts que tienen **permisos de ejecucion**. Se puede desactivar un script con `chmod -x`.
</details>

## Ejercicio 5
**El sistema no arranca y muestra el prompt `grub rescue>`. Que pasos seguirias para recuperar el arranque?**

<details>
<summary>Ver respuesta</summary>

Desde el prompt `grub rescue>`:

```
# 1. Listar particiones disponibles
grub rescue> ls

# 2. Buscar la particion que contiene /boot/grub
grub rescue> ls (hd0,1)/boot/grub
# (probar con diferentes particiones hasta encontrar los archivos)

# 3. Establecer el prefijo y la raiz
grub rescue> set prefix=(hd0,1)/boot/grub
grub rescue> set root=(hd0,1)

# 4. Cargar el modulo normal
grub rescue> insmod normal

# 5. Arrancar el modo normal de GRUB
grub rescue> normal
```

Esto restaura temporalmente el menu de GRUB. Una vez arrancado el sistema, se debe reinstalar GRUB permanentemente:

```bash
sudo grub-install /dev/sda
sudo update-grub
```
</details>

## Ejercicio 6
**Cual es la diferencia entre GRUB Legacy y GRUB2 en cuanto a la numeracion de particiones? Da un ejemplo concreto.**

<details>
<summary>Ver respuesta</summary>

La diferencia principal en la numeracion:

- **GRUB Legacy**: numera las particiones **desde 0**
  - `(hd0,0)` = primera particion del primer disco
  - `(hd0,1)` = segunda particion del primer disco

- **GRUB2**: numera las particiones **desde 1**
  - `(hd0,1)` = primera particion del primer disco
  - `(hd0,2)` = segunda particion del primer disco

En ambos casos, **los discos se numeran desde 0** (hd0, hd1, hd2...).

Ejemplo concreto: `/dev/sda1` (primera particion del primer disco)
- GRUB Legacy: `(hd0,0)`
- GRUB2: `(hd0,1)` o `(hd0,msdos1)` para MBR, `(hd0,gpt1)` para GPT
</details>

## Ejercicio 7
**Necesitas instalar GRUB2 en un sistema UEFI. Que comando usarias? Que particion es necesaria y que formato debe tener?**

<details>
<summary>Ver respuesta</summary>

Comando para instalar GRUB2 en UEFI:

```bash
grub-install --target=x86_64-efi --efi-directory=/boot/efi
```

Requisitos:
- Se necesita una **ESP (EFI System Partition)** en el disco.
- La ESP debe estar formateada en **FAT32**.
- Normalmente se monta en **`/boot/efi`**.
- Tamano recomendado: **100 - 550 MB**.
- GRUB instalara su cargador como un archivo `.efi` dentro de la ESP, tipicamente en `/boot/efi/EFI/<distro>/grubx64.efi`.

Para verificar las entradas de arranque UEFI se puede usar:
```bash
efibootmgr -v
```
</details>

## Ejercicio 8
**Describe que variable de `/etc/default/grub` modificarias en cada uno de estos escenarios:**

**a) Quieres que el sistema arranque automaticamente sin esperar, con timeout 0.**
**b) Quieres pasar el parametro `nomodeset` al kernel en todas las entradas.**
**c) Quieres que GRUB recuerde la ultima entrada seleccionada.**

<details>
<summary>Ver respuesta</summary>

**a) Timeout 0 (arranque inmediato):**
```
GRUB_TIMEOUT=0
```
Nota: Si GRUB_TIMEOUT_STYLE es "hidden", el menu no se mostrara. Se puede mantener pulsado Shift (BIOS) para forzar su aparicion.

**b) Parametro `nomodeset` para todas las entradas:**
```
GRUB_CMDLINE_LINUX="nomodeset"
```
Se usa `GRUB_CMDLINE_LINUX` porque afecta a **todas** las entradas del menu (incluyendo las de recuperacion). Si solo se quisiera para la entrada por defecto, se usaria `GRUB_CMDLINE_LINUX_DEFAULT`.

**c) Recordar la ultima entrada seleccionada:**
```
GRUB_DEFAULT=saved
GRUB_SAVEDEFAULT=true
```
Con `saved`, GRUB guarda la ultima seleccion y la usa como entrada por defecto en el siguiente arranque.

**Importante**: Despues de cualquiera de estos cambios, ejecutar `update-grub` o `grub-mkconfig -o /boot/grub/grub.cfg`.
</details>
