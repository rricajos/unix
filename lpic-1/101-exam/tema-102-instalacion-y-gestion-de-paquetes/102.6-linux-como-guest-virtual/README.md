# 102.6 - Linux como sistema guest de virtualizacion

## Peso: 1

## Objetivo del examen

Comprender las implicaciones de la virtualizacion y la computacion en la nube en un sistema guest Linux. Esto incluye comprender los conceptos generales de maquinas virtuales y contenedores, asi como las herramientas utilizadas para gestionar maquinas virtuales.

## Conocimientos clave

- Comprender los conceptos generales de maquinas virtuales y contenedores
- Comprender los elementos comunes de las maquinas virtuales en entornos IaaS
- Conocer las caracteristicas unicas de un sistema Linux al ejecutarse como guest virtual
- Conocer las herramientas de agente/invitado para maquinas virtuales (guest additions/tools)
- Comprender los problemas de clonacion de VMs (IDs unicos, MAC addresses)
- Conocer D-Bus y su papel en la comunicacion con el hipervisor

## Archivos, terminos y utilidades

- Maquina virtual (VM)
- Contenedor Linux
- Hipervisor (Tipo 1 y Tipo 2)
- Virtualizacion completa
- Paravirtualizacion
- KVM, Xen, VirtualBox, VMware
- Docker, LXC
- IaaS, PaaS, SaaS
- D-Bus
- `cloud-init`
- Guest additions / VMware Tools / QEMU Guest Agent
